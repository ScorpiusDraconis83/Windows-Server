
## **Wireshark: A Simple and Effective Method for Analyzing DNS Traffic**

Wireshark provides an easy and efficient way to analyze DNS traffic. Let’s understand how to capture and analyze DNS packets using Wireshark through an example.

### **Story: Nikhil’s DNS Traffic Analysis**

#### **Characters:**

* **Nikhil**: A tech enthusiast who wants to understand his network activities.
* **Wireshark**: A powerful tool that helps view and understand network traffic.

#### **Event:**

Nikhil is sitting at his computer and wants to know what happens when he visits a website.

1. **Starting Wireshark**:

   * Nikhil opens Wireshark and selects his network interface. He is ready to capture DNS traffic.

2. **Generating DNS Query**:

   * Nikhil types `www.example.com` in his browser. This generates a DNS query that asks what the IP address of `www.example.com` is.

3. **Capturing DNS Traffic in Wireshark**:

   * Wireshark captures the DNS traffic. Nikhil applies a filter: `dns` or `udp.port == 53`, so he can view only DNS packets.

4. **Analyzing DNS Packets**:

   * Nikhil sees several packets in Wireshark. He clicks on a packet labeled "Standard query". This packet contains the following information:

     * **Transaction ID**: Identifies the query.
     * **Flags**: Confirms whether the query is valid.
     * **Questions**: Shows which domain (e.g., `www.example.com`) Nikhil asked for the IP address.

5. **Response from DNS Server**:

   * After a while, Nikhil sees another packet labeled "Standard query response", which contains the IP address for `www.example.com`, such as `93.184.216.34`. This information comes from the DNS server.

6. **Caching**:

   * Wireshark also shows that this IP address is stored in the cache, so the DNS server won’t need to be queried again if someone visits `www.example.com` later.

7. **Identifying Issues**:

   * If Nikhil notices delays in the DNS response or strange packets, he can deduce that there might be a network issue, such as a "Man-in-the-Middle" attack.

### **Conclusion**

Through this story, Nikhil learns how DNS traffic works and how to view it using Wireshark. Wireshark not only helps capture DNS queries and responses but also helps in identifying potential issues in the network. By analyzing DNS traffic, users can ensure their network is secure and efficient.

## **Filtering UDP Traffic by Port in Wireshark**

In Wireshark, you can filter packets based on UDP port numbers to focus on specific traffic. Here's how to use this filter effectively:

### **Filtering UDP Traffic by Port**

1. **Filter for a Specific UDP Port**:
   If you want to filter packets for a specific UDP port (e.g., port 53 for DNS), use the following filter:

   ```
   udp.port == 53
   ```

2. **Filter for All UDP Traffic**:
   If you want to display all UDP traffic, simply use:

   ```
   udp
   ```

   This will show all packets using the UDP protocol.

3. **Filter for Multiple Specific UDP Ports**:
   If you want to filter for multiple specific UDP ports, use the `||` (OR) operator. For example, to filter for both DNS (port 53) and DHCP (port 67):

   ```
   udp.port == 53 || udp.port == 67
   ```
