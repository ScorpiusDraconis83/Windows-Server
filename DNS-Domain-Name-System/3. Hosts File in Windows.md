
# What is the Hosts File?

The **hosts file** is a plain text file used by operating systems to map hostnames to IP addresses. It acts as a local DNS system, allowing the machine to resolve domain names to IP addresses without needing to contact external DNS servers.

---

## Purpose and Functionality

The hosts file plays a vital role in managing network behavior and can be used for the following purposes:

1. **Local Website Testing**
   Developers can map a domain name to a local server's IP address to test websites before going live.

2. **Blocking Malicious Websites**
   By redirecting harmful domains to non-existent or local IP addresses, access to those websites can be effectively blocked.

3. **Speeding Up Name Resolution**
   By bypassing external DNS lookups, the system can resolve domain names more quickly.

4. **Redirecting Domains**
   You can redirect a domain to a specific IP address, which is useful for managing local development environments or custom routing.

---

## Example of a Hosts File Entry

Here’s what a typical hosts file entry looks like:

```
127.0.0.1       localhost
192.168.1.10    mylocalwebsite.com
```

* `127.0.0.1` is the loopback IP address, representing the local machine (localhost).
* `192.168.1.10` is a local server IP address mapped to a custom domain name.

---

## Location of the Hosts File (By Operating System)

* **Windows**: `C:\Windows\System32\drivers\etc\hosts`
* **Linux**: `/etc/hosts`
* **macOS**: `/etc/hosts`

---

## How to Edit the Hosts File on Windows

### Step-by-Step Instructions

1. **Open Notepad as Administrator**

   * Click the **Start** button and type **Notepad**.
   * Right-click on Notepad and choose **Run as administrator**.

2. **Open the Hosts File**

   * In Notepad, go to **File > Open**.
   * Navigate to: `C:\Windows\System32\drivers\etc`
   * Change the file type to **All Files (*.*)** to view the hosts file.
   * Select the `hosts` file and click **Open**.

3. **Edit the Hosts File**

   * Add new entries at the bottom in the format:

     ```
     [IP_Address] [Domain_Name]
     ```

     Example:

     ```
     192.168.1.10 mylocalwebsite.com
     ```

4. **Save Your Changes**

   * Click **File > Save**.
   * Close Notepad.

5. **Test the Changes**

   * Open a web browser and enter the domain name you added.
   * If everything is correct, the domain should resolve to the IP you specified.

---

## Flushing DNS Cache (To Apply Hosts File Changes)

After editing the hosts file, it's recommended to flush your DNS cache so changes take effect immediately.

### Steps to Flush DNS:

1. **Open Command Prompt as Administrator**

   * Click the Start button, type `cmd`, right-click on **Command Prompt**, and select **Run as administrator**.

2. **Run DNS Flush Command**

   * In the Command Prompt, type:

     ```
     ipconfig /flushdns
     ```
   * Press **Enter**.

This will clear your DNS cache and apply the new mappings from your hosts file.

---

## Important Notes

* Always **back up** the original hosts file before editing.
* If you can't save the file, ensure:

  * You have administrator rights.
  * The file is not set to **read-only**.
* The hosts file only affects **your local machine**, not other users or devices on the same network.
