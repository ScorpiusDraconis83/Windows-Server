### Website Binding with IIS

#### **IIS (Internet Information Services)**

IIS, or **Internet Information Services**, is a web server developed by Microsoft. It runs on Windows operating systems and is used to host websites. With IIS, you can make your website accessible over the internet or within a local network.

---

#### **Installing IIS**

To install IIS, follow these steps:

1. **Click on the Start Button:** Click on the Start button of your computer.
2. **Open Control Panel:** Type "Control Panel" and open it.
3. **Go to Programs and Features:** Click on "Programs" and then select "Programs and Features."
4. **Turn Windows Features On or Off:** Click on "Turn Windows features on or off."
5. **Select IIS:** Check the box next to "Internet Information Services" and click OK. This will install IIS on your computer.

---

### Setting Up a Website in IIS

Setting up a website in IIS is simple. Here’s a step-by-step process:

1. **Open IIS Manager:**

   * Click on the Start button and type "IIS." Then, click on "Internet Information Services (IIS) Manager."

2. **Add a New Site:**

   * In IIS Manager, right-click on "Sites" on the left panel and select "Add Website."

3. **Set Website Name and Path:**

   * A window will appear asking you to provide a name for your site (like "MyWebsite") and select the folder where your website files are located. For example, `C:\inetpub\wwwroot\MyWebsite`.

4. **Set Binding Options:**

   * Here, you will need to set the binding:

     * **IP Address:** Enter the specific IP address (e.g., `192.168.144.36`).
     * **Port:** Typically, HTTP uses port 80 and HTTPS uses port 443. You can also set a custom port (e.g., 8080).
     * **Host Name:** If you're using a domain name, enter it here (e.g., `mywebsite.com`).

5. **Create the Site:** After filling in all the details, click "OK." Your website is now set up in IIS!

---

### Types of Website Binding

IIS allows you to bind websites in two primary ways:

#### **1. Standard Way**

* In this method, each website uses a unique IP address and port.
* Example:

  * Default website: `http://192.168.144.36`
  * Site 1: `http://192.168.144.36/site1`

#### **2. Non-Standard Way**

* In this method, the same IP address is used with different ports.
* Example:

  * Default website: `http://192.168.144.36`
  * Site 1: `http://192.168.144.36:8080`
  * Site 2: `http://192.168.144.36:8081`
  * Site 3: `http://192.168.144.36:8082`

---

### Self-Signed Certificate

When using the HTTPS protocol, an SSL certificate is required. A **Self-Signed Certificate** is easy to create and is suitable for testing purposes.

#### Steps to Create a Self-Signed Certificate:

1. **Open IIS Manager.**
2. **Select Server Certificates:** Double-click on your server name and go to "Server Certificates."
3. **Choose Create Self-Signed Certificate:** On the right, click "Create Self-Signed Certificate."
4. **Name the Certificate:** Give your certificate a name (e.g., "MySelfSignedCert") and click OK.

**HTTPS URL Examples:**

* Without a port: `https://192.168.144.36`
* With a custom port: `https://192.168.144.36:8080`
