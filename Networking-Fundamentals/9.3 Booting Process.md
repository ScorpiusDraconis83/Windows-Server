
## ⚙️ **Understanding the Booting Process**

The **booting process** is the step-by-step startup routine a computer follows to load the **operating system** into memory and get the system ready for use. It’s like a backstage process that brings your machine to life every time you hit the power button.

---

## 🚀 **Stages of the Boot Process**

---

### 🔌 **1. Power Supply (SMPS – Switched Mode Power Supply)**

* Converts the AC power from the wall socket into **regulated DC power** required by internal components.
* Delivers power to the **motherboard, CPU, RAM, storage drives, GPU**, and connected devices.
* Crucial in every computing device — from everyday desktops to power-hungry gaming rigs and servers.

---

### 🧠 **2. BIOS (Basic Input/Output System)**

* Stored in ROM/Flash memory, BIOS firmware runs **immediately after power-on**.

  **Major Tasks Performed:**

  * **POST (Power-On Self Test):** Checks key hardware like RAM, processor, keyboard, and disk drives for proper functioning.
  * **Device Initialization:** Sets up hardware components for OS handover.
  * **Boot Sequence Selection:** Chooses the appropriate bootable device (HDD, SSD, USB, etc.) based on boot priority.

---

### 🥾 **3. Bootloader**

* A lightweight program designed to **load the operating system kernel** into system memory.

  **Popular Bootloaders:**

  * **GRUB** – Common in Linux distributions.
  * **Windows Boot Manager** – Default in Windows environments.

  **Key Functions:**

  * Loads and hands over control to the OS kernel.
  * Displays boot options or OS choices when multiple systems or kernels are installed.

---

### 🧬 **4. Kernel Loading**

* The **OS kernel** is now loaded into RAM.
* The kernel is the **core** of the operating system that interacts directly with hardware.

  **Kernel Tasks Include:**

  * Managing memory, CPU scheduling, and system processes.
  * Loading required **device drivers**.
  * Mounting the **root file system** (e.g., `/` in Linux).

---

### 🔁 **5. Init System (e.g., systemd)**

* Once the kernel is running, it launches the first user-space process.

  **Variants:**

  * **init** – Traditional Unix/Linux system initializer.
  * **systemd** – Modern init system in most current Linux distributions.

  **Responsibilities:**

  * Starts essential system services (networking, login, sound, logging).
  * Manages **targets** like multi-user, graphical mode, etc.

---

### 🔐 **6. Login Manager**

* The interface responsible for **user authentication and session management**.

  **Types:**

  * **Graphical (GUI):** GDM, SDDM, LightDM, etc.
  * **Text-based (CLI):** Login prompts on terminal screens.

  **Duties:**

  * Authenticates users via login credentials.
  * Initiates desktop environments (like GNOME, KDE) or terminal shells.
