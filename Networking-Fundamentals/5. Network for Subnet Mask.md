
## 🛡️ **Subnet Mask / Netmask / Network Mask**

A **Subnet Mask** (also known as **Netmask** or **Network Mask**) is like a filter that separates the **network part** from the **host part** of an IP address. It's essential in networking to define **which IPs belong to the same network**, and which are separate.

---

### 🧠 **Basic Structure of a Subnet Mask**

* **Total Length:** 32 bits (just like an IPv4 address)
* **Divided Into:** 4 blocks (called octets)
* **Each Block Contains:** 8 bits
* **Format:** Can be written in Binary or Decimal
  👉 Example: `11111111.00000000.00000000.00000000` = `255.0.0.0`

---

### 🧬 **Binary Logic Behind Subnet Masks**

* **Network Portion:** All bits are set to `1`
* **Host Portion:** All bits are set to `0`
  This combination tells devices which part of the IP is the **network ID** and which part refers to **specific devices** within that network.

---

### 🧾 **Subnet Masks by IP Class**

| **IP Class** | **Example IP** | **Structure** | **Binary Mask**                       | **Decimal Mask** |
| ------------ | -------------- | ------------- | ------------------------------------- | ---------------- |
| Class A      | `1.0.0.1`      | N.H.H.H       | `11111111.00000000.00000000.00000000` | `255.0.0.0`      |
| Class B      | `128.0.0.1`    | N.N.H.H       | `11111111.11111111.00000000.00000000` | `255.255.0.0`    |
| Class C      | `192.0.0.1`    | N.N.N.H       | `11111111.11111111.11111111.00000000` | `255.255.255.0`  |

🧩 **Legend:**
`N = Network bits`, `H = Host bits`
