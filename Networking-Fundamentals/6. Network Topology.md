
### Network Topology

Network topology is the physical and logical arrangement of devices (nodes) in a network, showing how they are connected and how data flows between them.

---

### 1. **Bus Topology**

* **Definition:** All devices are connected to a single central cable called the bus.
* **Example:** In a small office, all computers connect to one coaxial cable.
* **Working:** Data sent by one device travels along the bus to reach the destination.
* **Pros:** Easy to install and requires less cable.
* **Cons:** If the main cable (bus) fails, the entire network stops working.

---

### 2. **Star Topology**

* **Definition:** All devices are connected to a central hub or switch.
* **Example:** In a home network, all devices connect to a central Wi-Fi router.
* **Working:** Data passes through the central hub before reaching the destination device.
* **Pros:** Easy to manage and troubleshoot; failure of one device doesn’t affect others.
* **Cons:** If the central hub fails, the entire network goes down.

---

### 3. **Ring Topology**

* **Definition:** Each device is connected to exactly two other devices, forming a circular data path.
* **Example:** A campus network where buildings are connected in a ring.
* **Working:** Data travels in one direction around the ring.
* **Pros:** Data packets travel at high speed.
* **Cons:** Failure in any connection can disrupt the entire network unless there is a backup path.

---

### 4. **Mesh Topology**

* **Definition:** Every device is connected to every other device.
* **Example:** A corporate network where departments are interconnected.
* **Working:** Multiple paths exist for data to travel; if one path fails, data can take another.
* **Pros:** Highly reliable and fault-tolerant.
* **Cons:** Complex and costly to install due to many connections.

---

### 5. **Tree Topology**

* **Definition:** Combination of star and bus topologies; groups of star-configured networks connected via a bus.
* **Example:** Large organizations where departments branch off from a central root.
* **Working:** Hierarchical data flow from the root to branches.
* **Pros:** Scalable and easy to manage.
* **Cons:** If the backbone fails, entire branches can be affected.

---

### 6. **Hybrid Topology**

* **Definition:** A combination of two or more different topologies.
* **Example:** A company using star topology in one department and mesh in another.
* **Pros:** Flexible and scalable to meet varied needs.
* **Cons:** Complex design and implementation.

### Summary Examples of Network Topologies:

| Topology | Example Use Case                                   |
| -------- | -------------------------------------------------- |
| Bus      | Small office with computers on a single cable      |
| Star     | Home network with devices connected to Wi-Fi       |
| Ring     | School network with circular device connections    |
| Mesh     | ISP network with multiple interconnected nodes     |
| Tree     | Corporate network with central server and branches |
| Hybrid   | Large company mixing star, mesh, and ring          |
