### Classful IP Addressing

**Definition:**
Classful IP addressing was the original method of dividing the IP address space into different classes based on predefined ranges. This approach categorizes IP addresses into five classes (A, B, C, D, and E), each with a specific range and intended use.

**Classes:**

**Class A**

* **Range:** 1.0.0.0 to 126.255.255.255
* **Network/Host:** Division: 8 bits for network, 24 bits for host
* **Example:** 10.0.0.1 (Network: 10, Host: 0.0.1)

**Class B**

* **Range:** 128.0.0.0 to 191.255.255.255
* **Network/Host:** Division: 16 bits for network, 16 bits for host
* **Example:** 172.16.0.1 (Network: 172.16, Host: 0.1)

**Class C**

* **Range:** 192.0.0.0 to 223.255.255.255
* **Network/Host:** Division: 24 bits for network, 8 bits for host
* **Example:** 192.168.0.1 (Network: 192.168.0, Host: .1)

**Class D**

* **Range:** 224.0.0.0 to 239.255.255.255
* **Purpose:** Multicasting (not used for regular IP addresses)

**Class E**

* **Range:** 240.0.0.0 to 255.255.255.255
* **Purpose:** Experimental (reserved for future use)

**Example:**

* **IP Address:** 192.168.1.1
* **Class:** C
* **Subnet Mask:** 255.255.255.0

### Classless IP Addressing

**Definition:**
Classless IP addressing, also known as Classless Inter-Domain Routing (CIDR), was introduced to overcome the limitations of classful addressing. It allows for more flexible allocation of IP addresses by using variable-length subnet masks (VLSM). This method avoids the rigid boundaries of classful addressing.

**CIDR Notation:**
In CIDR, IP addresses are expressed with a prefix length, which indicates the number of bits used for the network portion of the address.

**Example:**

* **IP Address with CIDR Notation:** 192.168.1.0/24

  * **Network Portion:** 192.168.1.0
  * **Prefix Length:** /24 (means the first 24 bits are used for the network part)
  * **Subnet Mask:** 255.255.255.0

**Benefits:**

* More efficient use of IP address space.
* Flexibility to define network sizes.
* Improved routing efficiency.

### Classful IP Addressing

Classful IP addressing is an older method of dividing IP addresses into different classes. In this approach, IP address space is divided into blocks of different sizes:

**Fixed Classes:**

* **Class A:** For large networks, default subnet mask 255.0.0.0.
* **Class B:** For medium-sized networks, default subnet mask 255.255.0.0.
* **Class C:** For small networks, default subnet mask 255.255.255.0.
* **Class D:** Reserved for multicast.
* **Class E:** Reserved for experimental purposes.

**Subnet Mask:**

Each class has a fixed subnet mask, limiting flexibility in subnetting. For example, Class A always uses 255.0.0.0, Class B uses 255.255.0.0, and Class C uses 255.255.255.0.

**Address Range:**

Each class has a predefined IP address range. This can lead to inefficiencies, as the address blocks are fixed in size.

**Network/Host Division:**

The number of bits used for the network and host portions is fixed. In Class A, 8 bits are used for the network and 24 bits for the host; in Class B, 16 bits are used for the network and 16 bits for the host; and in Class C, 24 bits are used for the network and 8 bits for the host.

### 1. **What is IANA?**

**IANA (Internet Assigned Numbers Authority)** is the organization responsible for managing the global allocation of IP addresses and domain names. IANA controls large blocks of IP addresses globally.

**Example:** Suppose IANA has allocated a new IP address range, like 203.0.113.0/24.

---

### 2. **Regional Internet Registries (RIRs)**

The IP addresses allocated by **IANA** are assigned to **Regional Internet Registries (RIRs)**, which manage IP addresses in specific countries or regions.

**Example:** An RIR like APNIC might allocate the range 203.0.113.0/24 to an ISP in Asia.

---

### 3. **Internet Service Providers (ISPs)**

IP addresses allocated by **RIRs** are distributed to **Internet Service Providers (ISPs)**, who, in turn, provide these IP addresses to individuals and organizations.

**Example:** An ISP like BSNL might assign the 203.0.113.0/24 range to its customers.

---

### 4. **Local Networks and Devices**

The IP addresses provided by **ISPs** are used in **local networks** (like home or office networks). Your router and other devices within your network use these IP addresses.

**Example:** Your home router might assign 203.0.113.5 to your computer.

---

### 5. **Data Transmission**

When you visit a website, your computer sends a request. This request travels from your router to the ISP, then across the internet to the website server. The website server then sends data back to your computer, and you can view the site.

**Example:** You visit "example.com". Your computer sends a request, which travels through the ISP to the website server, and the website data is displayed in your browser.

---

### In Summary

1. **IANA** controls large blocks of IP addresses.
2. **RIRs** assign IP addresses to regional ISPs.
3. **ISPs** provide IP addresses to local networks and users.
4. **Local Networks** use these IP addresses to route and manage data.
