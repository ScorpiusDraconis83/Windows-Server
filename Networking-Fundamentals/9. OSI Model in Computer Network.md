
## 🧱 **OSI Model (Open Systems Interconnection)**

The OSI Model acts as a blueprint for how data flows across a network. It divides network communication into **seven layers**, ensuring seamless interaction among diverse systems and hardware across the globe.

---

### 🔹 **7. Application Layer**

This is the layer closest to the user — it's the face of network communication. It allows users to access services like email, browsing, and file transfers directly.

**Common Application Layer Protocols:**

* **HTTP / HTTPS** – Web communication (Ports: 80, 443)
* **FTP** – File transfers (Ports: 20, 21)
* **SMTP** – Sending emails (Port: 25)
* **DNS** – Domain name resolution (Port: 53)
* **SSH** – Secure remote login (Port: 22)
* **Telnet** – Remote shell access (Port: 23)
* **POP3 / IMAP** – Receiving emails (Ports: 110, 143)

---

### 🔸 **6. Presentation Layer**

This layer acts as the **translator** for the network. It converts data into readable formats, handles encryption/decryption, and compresses or decompresses content.

**Presentation Layer Examples:**

* **JPEG / MPEG** – Media compression
* **ASCII** – Standard character encoding
* **MIME** – Multimedia in emails
* **XDR** – Data formatting for cross-platform communication

---

### 🔹 **5. Session Layer**

The Session Layer keeps track of **who’s talking to whom**. It creates and maintains sessions (temporary dialogs) between devices, ensuring that communication starts, stays stable, and ends cleanly.

**Session Layer Protocols:**

* **L2TP / PPTP** – VPN tunneling protocols
* **NetBIOS** – File/Printer sharing over LAN
* **PAP** – Authentication during login sessions

---

### 🔸 **4. Transport Layer**

This layer ensures that data moves **smoothly and accurately**. It handles reliability, error checking, and flow control — making sure messages aren’t lost or jumbled.

**Key Protocols:**

* **TCP** – Reliable, ordered delivery
* **UDP** – Lightweight, faster delivery without guarantees

---

### 🔹 **3. Network Layer**

This layer is all about **routing and addressing**. It adds IP addresses and figures out the best route for the data to travel.

**Network Layer Protocols:**

* **IP** – Addressing and delivery
* **ICMP** – Sends error messages (like ping)
* **ARP** – Resolves IP to MAC
* **RIP** – Routing decisions based on hop count

---

### 🔸 **2. Data Link Layer**

Responsible for **framing, MAC addressing, and error detection**. It ensures that data packets are sent to the correct device on a local network and is the layer switches operate on.

**Data Link Layer Technologies:**

* **Ethernet (802.3)**
* **Wi-Fi (802.11)**
* **PPP**
* **HDLC**

---

### 🔹 **1. Physical Layer**

This layer is the actual **hardware level** — sending raw bits over wires, cables, and radio waves. It defines how devices physically transmit and receive data.

**Physical Layer Examples:**

* **Fiber Optics**
* **Copper Wires**
* **DSL Connections**

## Easy Table – All Layers at a Glance

| Layer No. | Layer Name   | Easy Job Explanation                    | Examples        |
| --------- | ------------ | --------------------------------------- | --------------- |
| 1         | Physical     | Sends 0s and 1s using wires and signals | Cables, NIC     |
| 2         | Data Link    | Sends data between nearby devices       | Switch, MAC     |
| 3         | Network      | Chooses path using IP address           | Router, IP      |
| 4         | Transport    | Delivers data safely in parts           | TCP, UDP        |
| 5         | Session      | Starts and ends connection              | API Session     |
| 6         | Presentation | Translates, encrypts, compresses data   | SSL, JPEG       |
| 7         | Application  | Used by apps like browser, email, etc.  | HTTP, SMTP, FTP |
