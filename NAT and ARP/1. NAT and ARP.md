
# Networking Essentials – NAT, ARP, Port Forwarding, Switch, and Proxy

---

## 1. NAT (Network Address Translation)

### What is NAT?

NAT is used to convert **private IP addresses** into a **public IP address**. It helps many devices in a home or office connect to the internet using just **one public IP**.

### Why is NAT used?

* **Because** there are **limited public IP addresses** available.
* It was needed to **allow multiple devices** to access the internet with **one IP**.

### Key Points:

* Saves public IPs by allowing IP sharing.
* Hides internal devices from the internet (adds security).

### Types of NAT:

| Type                           | Description                                                               |
| ------------------------------ | ------------------------------------------------------------------------- |
| Static NAT                     | One private IP is mapped to one public IP.                                |
| Dynamic NAT                    | Private IPs are mapped from a pool of public IPs.                         |
| PAT (Port Address Translation) | Many private IPs share one public IP using different ports. (Most common) |

### How NAT Works:

* Router gives **private IPs** to internal devices.
* Router has **one public IP** from the ISP.
* Router changes (translates) private IP to public when sending data.

## 2. ARP (Address Resolution Protocol)

### What is ARP?

ARP finds the **MAC address** of a device using its **IP address** inside a local network.

### Why is ARP used?

* Because devices need MAC addresses to **send data in a LAN**.
* IP address alone is not enough at the hardware (data link) level.

### How ARP Works:

1. Device checks if MAC is already saved (ARP cache).
2. If not, it sends an ARP Request to the LAN.
3. The correct device replies with its MAC address.

### ARP Cache:

Stores known IP–MAC pairs to avoid asking every time.

### Command:

```bash
arp -a
```

* Shows current ARP entries (IP to MAC).

### ARP in Wireshark:

* Shows ARP packets in real-time.
* Helps in analyzing who is talking to whom inside LAN.

## 3. Port Forwarding

### What is Port Forwarding?

Port forwarding sends data from the internet to a specific device inside a private network.

### Why is Port Forwarding used?

* Because private IPs are not directly reachable from the internet.
* Needed when we want to **access internal servers** (like CCTV, websites) from outside.

### Example:

* You want to access your home camera from your phone while away.
* Port forwarding helps by connecting your public IP to the camera’s private IP and port.

---

## 4. Switch vs Proxy

### Switch

* Connects devices in a local network (LAN).
* Uses **MAC addresses** to send data to the right device.

### Why use a Switch?

* To reduce network traffic and send data **only to the correct device**.
* Needed for fast and organized LAN communication.

### Proxy Server

* Works as a **middleman** between your device and the internet.
* Controls, filters, and monitors internet use.

### Why use a Proxy?

* To **block unwanted sites**, **save bandwidth**, or **hide identity**.
* Needed in schools, offices, and secure environments.

## Conclusion

| Technology      | What It Does                             | Why It Is Needed                                            |
| --------------- | ---------------------------------------- | ----------------------------------------------------------- |
| NAT             | Converts private IP to public IP         | To save public IPs and connect many devices to the internet |
| ARP             | Finds MAC address using IP               | Because data needs MAC to be sent in a local network        |
| Port Forwarding | Sends outside requests to inside devices | To access internal servers from outside                     |
| Switch          | Manages LAN traffic using MAC addresses  | To send data only to the right device                       |
| Proxy Server    | Filters and controls internet access     | To monitor, block, or hide internet activity                |
