
## 👥 User Accounts in Windows 11 Enterprise

| **Account Type**     | **Description**                                                                                                 |
| -------------------- | --------------------------------------------------------------------------------------------------------------- |
| **1. Administrator** | Full control over the system. Can install software, manage other user accounts, and change all system settings. |
| **2. Standard User** | Limited permissions. Can use most apps and change non-security-related settings that don’t affect other users.  |
| **3. Guest Account** | Temporary access without a password. Very restricted permissions, mainly for basic use.                         |

---

## 🧾 Creating User Accounts

| **Method**                    | **Steps / Description**                                                                                                       |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **1. Settings**               | Go to *Settings > Accounts > Family & other users*, click **Add someone else to this PC**. Create local or Microsoft account. |
| **2. Control Panel**          | Use the *User Accounts* section to create and manage accounts.                                                                |
| **3. Local Users and Groups** | Run `lusrmgr.msc` (available in Pro/Enterprise editions) for detailed user and group management.                              |

---

## ⚙️ Managing User Accounts

| **Task**                 | **Description**                                                                         |
| ------------------------ | --------------------------------------------------------------------------------------- |
| **Change Account Type**  | Switch between standard and administrator types from *Settings > Family & other users*. |
| **Set Passwords**        | Admins can set or reset passwords for other users.                                      |
| **Disable/Enable Users** | Accounts can be temporarily disabled to restrict access without deleting them.          |

---

## 🔐 User Permissions

| **Permission Type**            | **Description**                                                        |
| ------------------------------ | ---------------------------------------------------------------------- |
| **Read/Write Permissions**     | Controls if users can read or modify files and folders.                |
| **Administrative Permissions** | Only available to Administrator group members for system-wide changes. |

---

## 🛠 Using Local Users and Groups

| **Feature**                | **Description**                                                     |
| -------------------------- | ------------------------------------------------------------------- |
| **Access Tool**            | Run `lusrmgr.msc` from the Run dialog.                              |
| **Capabilities**           | View all user/group accounts, including hidden or disabled ones.    |
| **Manage User Properties** | Set password expiry, disable accounts, or modify group memberships. |

---

## 📂 Viewing User Groups

| **Group Name**     | **Purpose**                            |
| ------------------ | -------------------------------------- |
| **Administrators** | Full system control.                   |
| **Power Users**    | Limited administrative capabilities.   |
| **Guests**         | Restricted access for temporary users. |

---

## 💻 Command-Line Management

| **Command**      | **Function**                                     |
| ---------------- | ------------------------------------------------ |
| `net user`       | Lists all user accounts.                         |
| `whoami /groups` | Displays all groups the current user belongs to. |

---

## ✅ Conclusion

Effective user account management in **Windows 11 Enterprise** is essential for maintaining system **security** and **access control**. Tools like *Settings*, *Control Panel*, and *Local Users and Groups* enable administrators to create, modify, and secure user accounts. This structured approach ensures system integrity while allowing users to work efficiently.
