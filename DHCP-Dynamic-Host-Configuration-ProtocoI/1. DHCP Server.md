
**Dynamic Host Configuration Protocol (DHCP)** is a network management protocol that automatically assigns IP addresses and other network configuration details to devices connected to a network. It operates on a client–server architecture, simplifying and streamlining network administration.

### How DHCP Works

DHCP uses the **DORA** process—a four-step sequence—to allocate IP addresses to clients:

* **Discover**
* **Offer**
* **Request**
* **Acknowledgement**

Let’s break down each step with examples.

#### 1. Discover

* **What happens**: When a DHCP client (e.g., a laptop or smartphone) connects to the network, it broadcasts a **DHCP Discover** message to locate available DHCP servers.
* **Example**: Nikhil’s laptop sends:

  ```
  DHCP Discover: “Is there a DHCP server out there?”
  ```

#### 2. Offer

* **What happens**: Any DHCP server that receives the Discover reply sends back a **DHCP Offer** message. This includes a proposed IP address, subnet mask, lease duration, and other settings.
* **Example**: The DHCP server responds:

  ```
  DHCP Offer: “We can offer you IP address 192.168.1.10.”
  ```

#### 3. Request

* **What happens**: The client receives one or more offers, then broadcasts a **DHCP Request** message indicating which offer it accepts.
* **Example**: Nikhil’s laptop broadcasts:

  ```
  DHCP Request: “I accept 192.168.1.10.”
  ```

#### 4. Acknowledgement

* **What happens**: The DHCP server confirms the allocation with a **DHCP Ack** message, finalizing the IP lease and providing full configuration.
* **Example**: The server replies:

  ```
  DHCP Ack: “192.168.1.10 has been successfully assigned to you.”
  ```

---

### DHCP Lease Renewal

* Each IP lease has an expiration time (typically hours or days).
* Before expiration, the client unicasts a renewal **Request** to the original DHCP server.
* If the server agrees, it sends back an **Ack**.
* If the lease expires without renewal, the client must restart the DORA process.

---

## Key DHCP Components

* **DHCP Server**: Manages and assigns IP addresses and other network settings.
* **DHCP Client**: The device requesting an IP lease.
* **Address Pool**: A range of IPs the server can assign.
* **Lease**: The granted IP address and its valid duration.

---

## Benefits of DHCP

1. **Automation**: Eliminates manual IP configuration.
2. **Reduced Errors**: Avoids conflicts and typos in IP assignment.
3. **Efficient Use of Addresses**: Reclaims addresses when clients disconnect.
4. **Centralized Control**: Simplifies network-wide configuration changes.

---

## Why DHCP Matters

In large networks with hundreds of devices, DHCP is indispensable. It automates IP management, enhances reliability, and lets users connect without manual setup, boosting both efficiency and uptime.

## Capturing and Analyzing DHCP Traffic with Wireshark

You can use **Wireshark** to capture and inspect DHCP packets as they follow the DORA process.

### Steps to Capture DHCP Traffic

1. **Open Wireshark** and start a new capture on your network interface.
2. **Open a Command Prompt** on a Windows client.
3. **Release the current lease**:

   ```bash
   ipconfig /release
   ```
4. **Renew the lease**:

   ```bash
   ipconfig /renew
   ```
5. **Stop the Wireshark capture**—you’ll now have captured the Discover, Offer, Request, and Ack packets.

---

### Analyzing DHCP Packets

* **Filter DHCP traffic**: Use the display filter `bootp` in Wireshark.
* **Examine each packet**: Click through the packet list to view Ethernet, IP, UDP, and DHCP layers.
* **Inspect DHCP options**: Check details like lease time, subnet mask, gateway, and DNS servers.
