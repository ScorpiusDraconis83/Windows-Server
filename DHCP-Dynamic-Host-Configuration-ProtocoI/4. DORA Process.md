The **DORA process** is the sequence of steps used by the **Dynamic Host Configuration Protocol (DHCP)** to assign an **IP address** to a client device on a network. It consists of four key steps:  

---

### **1. Discover (D) – DHCP Discover**  
When a client (such as a computer, phone, or printer) connects to a network and needs an IP address, it sends a **DHCP Discover** message.  
- This is a **broadcast** message sent to the network because the client doesn’t yet have an IP address.  
- The message is sent to **255.255.255.255** or the subnet’s broadcast address.  
- It is looking for any available **DHCP server** to respond.  

---

### **2. Offer (O) – DHCP Offer**  
When a **DHCP server** receives the Discover message, it checks its pool of available IP addresses and sends a **DHCP Offer** back to the client.  
- This message contains:  
  - An **offered IP address**  
  - The **subnet mask**  
  - The **default gateway**  
  - The **DNS server address**  
  - The **lease time** (how long the IP address is valid)  
- This message is sent as a **unicast** or **broadcast**, depending on network settings.  

---

### **3. Request (R) – DHCP Request**  
The client receives **one or more** offers (if multiple DHCP servers respond) and selects one. It then sends a **DHCP Request** message back to the chosen DHCP server.  
- This message tells the server that the client **accepts** the offered IP address.  
- It also informs other DHCP servers (if applicable) that their offers were declined.  
- This step prevents multiple DHCP servers from assigning conflicting IP addresses.  

---

### **4. Acknowledge (A) – DHCP Acknowledge (ACK)**  
Finally, the DHCP server sends a **DHCP Acknowledgment (ACK)** to confirm the lease of the IP address to the client.  
- The client can now use the assigned IP address.  
- The DHCP lease **duration** starts at this point.  
- If the server cannot allocate the requested IP (e.g., it was assigned to another client), it may send a **DHCP NAK (Negative Acknowledgment)** instead, forcing the client to restart the DORA process.  

---

### **Summary of the DORA Process**  
1️⃣ **Discover** – Client broadcasts a request for an IP address.  
2️⃣ **Offer** – DHCP server responds with an available IP address.  
3️⃣ **Request** – Client requests the offered IP address.  
4️⃣ **Acknowledge** – Server confirms and assigns the IP.  

---
DORA प्रक्रिया नेटवर्किंग में **DHCP (Dynamic Host Configuration Protocol)** के माध्यम से **IP Address** असाइन करने की प्रक्रिया को दर्शाती है। यह चार चरणों में होती है:  

---

### **1. Discover (D) – Client द्वारा सर्वर की खोज**  
जब कोई नया डिवाइस (जैसे कंप्यूटर, मोबाइल, या प्रिंटर) नेटवर्क से कनेक्ट होता है और उसे IP Address की आवश्यकता होती है, तो वह **DHCP Discover** संदेश भेजता है।  

🔹 यह एक **ब्रॉडकास्ट मैसेज** होता है, क्योंकि क्लाइंट के पास अभी कोई IP Address नहीं होता।  
🔹 यह संदेश नेटवर्क में सभी उपलब्ध **DHCP Servers** को भेजा जाता है ताकि वे प्रतिक्रिया दें।  

📌 **उदाहरण:** जब आप अपने लैपटॉप को Wi-Fi से कनेक्ट करते हैं, तो यह स्वचालित रूप से एक IP Address के लिए अनुरोध करता है।  

---

### **2. Offer (O) – DHCP Server का Response**  
नेटवर्क में उपलब्ध **DHCP Server** इस Discover संदेश को प्राप्त करता है और **DHCP Offer** संदेश भेजता है।  

🔹 इस संदेश में निम्नलिखित जानकारी होती है:  
- एक **उपलब्ध IP Address**  
- **Subnet Mask**  
- **Default Gateway**  
- **DNS Server का पता**  
- **Lease Time** (IP Address कितनी देर तक मान्य रहेगा)  

🔹 यह मैसेज आमतौर पर **Unicast** या **Broadcast** के रूप में भेजा जाता है।  

📌 **उदाहरण:** जब आप ऑफिस में नेटवर्क से जुड़ते हैं, तो DHCP Server एक उपयुक्त IP Address ऑफर करता है।  

---

### **3. Request (R) – Client द्वारा IP की मांग**  
DHCP Server से मिलने वाले एक या अधिक **DHCP Offer** में से क्लाइंट एक को चुनता है और **DHCP Request** संदेश भेजता है।  

🔹 इस संदेश में यह जानकारी होती है कि क्लाइंट किस DHCP Server के ऑफर को स्वीकार कर रहा है।  
🔹 यह अन्य DHCP Servers को भी संकेत देता है कि उनके ऑफर को अस्वीकार कर दिया गया है।  

📌 **उदाहरण:** यदि किसी नेटवर्क में कई DHCP Servers हैं, तो क्लाइंट एक उपयुक्त Server से IP Address प्राप्त करता है और बाकी ऑफर को अस्वीकार कर देता है।  

---

### **4. Acknowledge (A) – सर्वर द्वारा पुष्टि (Acknowledgment)**  
अब, DHCP Server क्लाइंट को **DHCP Acknowledge (ACK)** संदेश भेजता है, जिससे IP Address क्लाइंट को असाइन हो जाता है।  

🔹 इसमें क्लाइंट को सभी आवश्यक नेटवर्क सेटिंग्स प्रदान की जाती हैं।  
🔹 अब क्लाइंट इस IP Address का उपयोग नेटवर्क संचार के लिए कर सकता है।  
🔹 अगर सर्वर किसी कारण से IP Address असाइन नहीं कर सकता, तो वह **DHCP NAK (Negative Acknowledgment)** भेजता है, जिससे क्लाइंट को नया अनुरोध भेजना पड़ता है।  

📌 **उदाहरण:** जब आपका स्मार्टफोन Wi-Fi से कनेक्ट होता है, तो इसे एक मान्य IP Address मिलता है, और तब आप इंटरनेट ब्राउज़ कर सकते हैं।  

---

### **निष्कर्ष**  
💡 **DORA Process का सार:**  
1️⃣ **Discover** – क्लाइंट DHCP Server को खोजता है।  
2️⃣ **Offer** – DHCP Server उपलब्ध IP Address ऑफर करता है।  
3️⃣ **Request** – क्लाइंट उस IP Address के लिए अनुरोध करता है।  
4️⃣ **Acknowledge** – DHCP Server IP को असाइन कर देता है।  

**DORA प्रक्रिया** नेटवर्किंग में **IP Address को स्वचालित रूप से असाइन करने** का एक कुशल और संगठित तरीका है, जिससे नेटवर्क एडमिनिस्ट्रेशन आसान और प्रभावी बनता है।
