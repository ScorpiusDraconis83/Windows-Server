
## Follow these steps to configure a DHCP server:

### Step 1: Install the DHCP Server Role

1. **Open Server Manager**: Start → Administrative Tools → Server Manager
2. Click **Add Roles and Features**
3. Select **Role-based or feature-based installation** and click **Next**
4. Choose the server on which you want to install the DHCP Server
5. In **Server Roles**, check **DHCP Server** and click **Next**
6. Click **Add Features** to include any required features for the DHCP Server
7. Review your selections and click **Install**

### Step 2: Configure the DHCP Server

1. **Open the DHCP Management Console**: Start → Administrative Tools → DHCP

2. **Create a DHCP Scope**:

   * Right-click **DHCP** and select **New Scope**
   * Name the scope (e.g., “LAN Scope”)
   * Specify the IP address range (e.g., 192.168.1.10 – 192.168.1.200)
   * Enter the subnet mask (e.g., 255.255.255.0)
   * Set the lease duration (Default: 8 days)
   * Click **Next** and configure any additional scope options

3. **Configure DHCP Options**:

   * Under **DHCP → IPv4 → Server Options**, right-click and select **Configure Options**
   * In option 3 (Router), enter the default gateway’s IP address
   * In option 6 (DNS), enter the DNS server’s IP address
   * Configure any other required options and click **OK**

4. **Activate the DHCP Scope**:

   * Under **DHCP → IPv4 → Scopes**, right-click your new scope and select **Activate**

### Step 3: Configure DHCP Clients

1. **Open the Network Adapter Properties**:

   * Right-click the network connection and choose **Properties**
   * Select **Internet Protocol Version 4 (TCP/IPv4)** and click **Properties**

2. **Enable DHCP on the Client**:

   * Choose **Obtain an IP address automatically** and **Obtain DNS server address automatically**
   * Click **OK** and close the connection properties

3. **Renew the IP Address**:

   ```bash
   ipconfig /renew
   ```

---

### Additional Features

* **DHCP Server Logs**: You can configure DHCP server logs to record which IP addresses were assigned and track other activities.
* **DHCP Filtering**: The DHCP server can be set to allocate addresses only to clients with specified MAC addresses or client IDs.
* **DHCP Classes**: Use DHCP classes to assign different address pools to different types of clients.

---

### Conclusion

Configuring a DHCP server on Windows Server is straightforward. By installing and setting up the DHCP role, you can automatically distribute IP addresses and network settings to devices, making network management easier and more efficient.
