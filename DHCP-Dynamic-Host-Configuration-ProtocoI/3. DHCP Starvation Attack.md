
### What is DHCP?

**DHCP** (Dynamic Host Configuration Protocol) is a protocol that automatically assigns IP addresses to devices (like computers, printers) on a network. When you connect your computer to the network, a DHCP server gives it an IP address so it can communicate with the Internet or other devices on the network.

* **Ports used:**

  * Server listens on **UDP 67**
  * Client listens on **UDP 68**

---

### What is a DHCP Starvation Attack?

A **DHCP Starvation Attack** is a type of cyberattack in which an attacker exhausts all available IP addresses on a DHCP server. Once the pool is empty, no new devices can obtain an IP address and join the network.

---

### How Does This Attack Work?

1. **Attacker’s Action**

   * The attacker uses a tool to send a large number of DHCP Discover messages, each spoofing a different MAC address.
   * For example, the attacker might send 1,000 bogus requests, making the DHCP server believe 1,000 distinct devices are requesting addresses.

2. **DHCP Server’s Response**

   * The server replies to each Discover with a DHCP Offer, allocating IP addresses until its pool (e.g., 254 addresses) is exhausted.

3. **Result**

   * When a legitimate user tries to connect, the DHCP server has no addresses left, so that device cannot join the network.
   * This effectively creates a **Denial of Service (DoS)** condition for new clients.

---

### Simple Example

* **Office Setup**: Your office network is 192.168.1.0/24, so the DHCP server can hand out up to 254 addresses.
* **Attacker**: Uses a script to send thousands of DHCP Discover messages.
* **Server**: Assigns all 254 addresses to the attacker’s fake clients.
* **New Device**: When you plug in a new laptop, the server has no addresses left, so it cannot get online.

---

### Defenses Against DHCP Starvation

1. **Port Security**

   * Configure your switch so each port accepts only a fixed number of MAC addresses.

2. **DHCP Snooping**

   * Enable DHCP snooping on your switches so only trusted DHCP servers can respond to client requests.

3. **Network Monitoring**

   * Regularly monitor for unusually high volumes of DHCP traffic or repeated MAC-address changes.

---

### Types of Attacks Spawned by DHCP Starvation

1. **Denial of Service (DoS)**

   * **What it is**: Exhausting the DHCP pool so legitimate clients can’t get addresses.
   * **Impact**: New devices are prevented from joining the network.

2. **Man-in-the-Middle (MITM)**

   * **What it is**: After starvation, the attacker runs a rogue DHCP server.
   * **Impact**: Clients receive addresses (and other settings) from the attacker’s server, allowing traffic interception or redirection.

---

### Related Attacks

* **DHCP Spoofing**: Attacker sets up a fake DHCP server to hand out malicious configuration.
* **DHCP Relay Attack**: Attacker relays DHCP packets between networks to misuse the server’s resources.
